FLUXOS PRINCIPAIS DE DADOS
→ API Request Flow: Cliente → API Endpoint → ViewSet → Repository → Model → Django ORM → Database
→ Bulk Upload Flow: Cliente → Upload File → FileProcessor → Validation → Bulk Create → Database
→ Loan Creation: POST /api/loans/ → LoanViewSet.create() → Book.reserve_copy() → Update available_quantity → Save Loan
→ Loan Return: POST /api/loans/{id}/return_loan/ → Loan.mark_as_returned() → Book.return_copy() → Update available_quantity
→ Authentication: POST /api/auth/token/ → Validate credentials → Generate JWT → Return access + refresh tokens
→ Search Flow: GET /api/books/?search=query → ViewSet.filter_queryset() → Repository → Database Query → Paginated Response
RELACIONAMENTOS DO BANCO DE DADOS
Loan.book → Book (ForeignKey, on_delete=PROTECT) - Um empréstimo pertence a um livro
Loan.user → LibraryUser (ForeignKey, on_delete=PROTECT) - Um empréstimo pertence a um usuário
Book.loans (reverse relation) - Um livro pode ter múltiplos empréstimos ao longo do tempo
LibraryUser.loans (reverse relation) - Um usuário pode ter múltiplos empréstimos ao longo do tempo
PROTECT: Impede exclusão de livros/usuários com empréstimos ativos
